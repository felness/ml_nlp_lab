# Отчёт исследования: LLM Credit Scoring + Classical ML

## 1. Описание датасета

**UCI Adult (Census Income) — ID 2**

- **Источник:** UC Irvine Machine Learning Repository
- **Объём:** 48 842 записей, 14 признаков
- **Задача:** предсказать, превышает ли годовой доход $50K
- **Применение к кредитованию:** доход >$50K используется как прокси кредитоспособности — признак способности обслуживать долг

**Распределение классов:** 76% — доход ≤$50K (отказ), 24% — доход >$50K (одобрение). Датасет несбалансирован.

## 4. Оценка перформанса LLM (Qwen2.5:0.5B) на датасете UCI Adult

Оценка проводилась на 20 примерах из тестовой выборки. Признаки датасета преобразовывались в текстовые описания и передавались в `POST /analyze`. Ответ LLM сопоставлялся с истинной меткой.

- **Проверено примеров:** 20
- **Неопределённых решений (UNKNOWN):** 19 из 20 (95%)

| Метрика | Значение |
|---------|----------|
| Accuracy  | **0.7500** |
| Precision | **0.0000** |
| Recall    | **0.0000** |
| F1 Score  | **0.0000** |

```
                  precision    recall  f1-score   support

<=50K (REJECTED)       0.75      1.00      0.86        15
 >50K (APPROVED)       0.00      0.00      0.00         5

        accuracy                           0.75        20
       macro avg       0.38      0.50      0.43        20
    weighted avg       0.56      0.75      0.64        20
```

**Интерпретация:** Accuracy 0.75 — это ложная точность. Поскольку модель возвращала UNKNOWN (маппируется в REJECTED) в 95% случаев, она фактически предсказывала отказ всем. Из-за несбалансированности датасета (76% REJECTED) это дало высокую accuracy при нулевом F1 на классе APPROVED.

## 5. Выводы

1. **Qwen2.5:0.5B без дообучения непригоден для автономного кредитного скоринга.** 95% ответов — UNKNOWN. Модель понимает задачу и отвечает по-русски, но не стабильно следует формату вывода. F1 = 0.0 означает полный провал на классе APPROVED.

2. **GradientBoosting (F1 = 0.72, Accuracy = 0.88) — ожидаемо лучший** на данном датасете. Выбран как ML-инструмент для MCP-сервера.

3. **Связка LLM + MCP-инструменты** — правильный архитектурный выбор для PoC: LLM обрабатывает произвольный текстовый ввод и оркестрирует вызовы, ML-инструменты обеспечивают детерминированные числовые результаты.

4. **Ограничения прототипа:**
   - Датасет Adult — перепись населения, а не банковские кредитные данные
   - Qwen2.5:0.5B слишком мала для надёжного tool-calling
   - Отсутствует история платежей — ключевой признак в реальном скоринге

5. **Дальнейшие улучшения:**
   - Языковая модель ≥7B параметров для стабильного следования инструкциям(либо использования мощных моделей по api)
   - Специализированный кредитный датасет с историей платежей
   - SHAP-значения для объяснимости решений (регуляторный compliance)
   - Калибровка вероятностей и ансамблирование ML-моделей
